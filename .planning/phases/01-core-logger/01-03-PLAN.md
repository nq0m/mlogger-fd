Stats API endpoint design from RESEARCH.md Code Example:
- GET /api/stats returns: {total, raw_points, multiplier, score, rate_10min, rate_1hr, breakdown: {"20M_CW": 5, ...}}
- Rate 10min: COUNT qsos WHERE timestamp >= now - 10min, then × 6 for hourly rate
- Rate 1hr: COUNT qsos WHERE timestamp >= now - 1hr
- Multiplier: COUNT(DISTINCT band || '_' || mode) WHERE is_dupe = 0
- Score: raw_points × multiplier (Phase 1: no power multiplier, no bonuses)
- Breakdown: GROUP BY band, mode query

Rate meter windows per CONTEXT.md discretion: last 10 minutes, last 1 hour, overall session. Show current rate prominently, peak rate as secondary.

Points calculation from field-day-logger-plan.md §2.5:
- Raw Score = Sum of all QSO points (excluding dupes)
- Total Score = Raw Score × Multiplier (Phase 1 simplified — no power multiplier, no bonuses)
- Multiplier = count of unique band+mode combinations worked

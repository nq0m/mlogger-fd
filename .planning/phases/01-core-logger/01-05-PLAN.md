<interfaces>
Cabrillo format from field-day-logger-plan.md Appendix A:

Header lines:
```
START-OF-LOG: 3.0
CREATED-BY: FDLogger v1.0
ARRL-SECTION: NH
CALLSIGN: N0CALL
CONTEST: ARRL-FIELD-DAY
CATEGORY-OPERATOR: SINGLE-OP
CATEGORY-POWER: LOW
CATEGORY-STATION: PORTABLE
CLAIMED-SCORE: <calculated>
END-OF-LOG:
```

QSO line fixed-width format (from Appendix A table):
```
QSO: <freq 6> <mode 4> <date YYYY-MM-DD> <time HHMM> <sent-call 10+> <sent-exch 10> <rcvd-call 10+> <rcvd-exch 10>
```

Band-to-frequency mapping (standard Field Day):
160M→1800, 80M→3500, 40M→7000, 20M→14000, 15M→21000, 10M→28000, 6M→50000, 2M→144000, 70CM→432000

Mode-to-Cabrillo mapping:
CW→CW, SSB→PH, FM→FM, RTTY→RY, FT8→DG, FT4→DG, PSK31→DG

Dupe QSOs: replace "QSO:" prefix with "QSO: ---dupe---" (15 chars total, matches fixed-width).

From RESEARCH.md code example: use bytes.Buffer for building output, db.Query for QSOs, Content-Type: text/plain, Content-Disposition: attachment header.

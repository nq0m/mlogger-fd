## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Client → API /api/station-config | Untrusted client sends station config values; server validates before DB write |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | PUT /api/station-config handler | mitigate | Validate all fields via model.ValidateStationConfig() before DB write — rejects empty callsign, invalid class/section, out-of-range tx count, invalid power level. Returns 400 with error message. |
| T-02-02 | Tampering | SQL injection via station_config queries | mitigate | All queries use parameterized ? placeholders (INSERT OR REPLACE with ? params, SELECT with ? for id=1). No string interpolation. Same pattern as existing Phase 1 QSO handlers. |
| T-02-03 | Tampering | Config overwrite by unauthorized operator on LAN | accept | Trusted LAN per AGENTS.md — all operators have equal access. Station config is set once at contest start. If open WiFi vector is ever added, gate with shared-secret token check. |
| T-02-04 | Information Disclosure | Config exposed to LAN clients | accept | Intended behavior — station config is shared across operators. No secrets stored (callsign/class/section are public info). |
</threat_model>

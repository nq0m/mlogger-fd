Implement connectWebSocket() following RESEARCH.md Pattern 5 exactly:
- Derive WebSocket URL: const protocol = location.protocol === 'https:' ? 'wss:' : 'ws:'; const url = `${protocol}//${location.host}/ws`;
- Set shouldReconnect = true
- Inner connect() function: creates new WebSocket(url), assigns onopen, onmessage, onclose, onerror

- onerror: console.error only (onclose will fire after, reconnect handled there)

Implement disconnectWebSocket(): shouldReconnect = false; clearTimeout(reconnectTimer); ws.close(); ws = null; wsConnected = false.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser localStorage → Client code | Operator identity stored client-side; other browser tabs on same device could read it (same-origin) — acceptable for Field Day |
| WebSocket message → Client state | Browser receives server-generated JSON; no user input flows through WebSocket inbound |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-10 | Tampering | JSON injection via WebSocket | accept | All WebSocket messages are server-generated (broadcast only). JSON.parse() in browser is safe — does not evaluate code. Svelte template bindings escape HTML by default. |
| T-02-11 | Information Disclosure | Operator identity in localStorage | accept | Stored client-side only. Same-origin policy limits access to this application. Operator identity is a name/callsign — not a secret. Field Day operators share a tent; physical access is the primary threat model. |
| T-02-12 | Spoofing | Operator identity falsification | accept | No authentication — trusted LAN per AGENTS.md. An operator could change their localStorage value mid-session, but this is a feature (operators may swap radios). |
| T-02-13 | Denial of Service | WebSocket reconnect loop | mitigate | Fixed 2-second reconnect delay prevents tight loops. shouldReconnect flag enables clean disconnect. Browser throttles rapid reconnects natively. |
</threat_model>
